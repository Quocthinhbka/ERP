# Report — Cải thiện Frontend toàn diện

**Thời gian:** 2026-07-23 12:21:53

## Yêu cầu thay đổi

Thực hiện toàn bộ các điểm cải thiện đã phân tích cho Frontend (`apps/web`): bảo trì, bảo mật route, xử lý lỗi, performance, testing, cấu hình API port.

## Việc đã thực hiện

### Giai đoạn 1 — Quick wins
- **`src/lib/errors.ts`**: Hàm `getApiErrorMessage`, `getLoginErrorMessage` dùng chung; thay thế helper trùng lặp ở AccountsPage, EmployeeFormPage, OrganizationPage.
- **Thống nhất API port 3000**: `vite.config.ts`, `scripts/dev-*.sh`, e2e helpers (`e2e/api-base.ts`).
- **`ProtectedRoute` + `PageSpinner`**: Loading khi bootstrap auth thay vì màn hình trắng.
- **`DashboardPage`**: Hiển thị vai trò thực (`Quản trị hệ thống` / `Người dùng`) thay vì hardcode "Super Admin".
- **`PermissionRoute` + `ForbiddenPage`**: Guard route theo quyền trong `App.tsx`.

### Giai đoạn 2 — Cấu trúc
- **Tách `OrganizationPage`**: 10 module dưới `pages/setup/organization/`; page chính ~1100 dòng.
- **Lazy loading routes**: React.lazy cho setup pages và HR list/detail nặng.
- **`ErrorBoundary`**: Bọc `<MainLayout />` catch lỗi runtime.
- **`NotFoundPage`**: Catch-all `*` → 404 thay vì redirect Dashboard.

### Giai đoạn 3 — Chất lượng & scale
- **TanStack Query**: `@tanstack/react-query`, `QueryClientProvider` trong `main.tsx`; refactor `EmployeesPage`, `PermissionsPage`, `useEmployeeProfileLayout`.
- **Refresh token queue**: Singleton promise trong `lib/api.ts` tránh refresh song song.
- **Personal/HR mode**: Wrapper pages (`PersonalProfileDetailPage`, `HrEmployeeDetailPage`, …) truyền prop `mode` thay vì parse pathname.
- **Vitest**: `vitest.config.ts`, unit tests `errors.test.ts`, `employee-profile-form.test.ts`.
- **Layout tokens**: `src/styles/layout-tokens.ts`.
- **data-testid**: employees-page, employee-create-btn, org-tree, permissions-page, …
- **Error handling**: Catch + message trên EmployeesPage, PermissionsPage, profile layout hook.
