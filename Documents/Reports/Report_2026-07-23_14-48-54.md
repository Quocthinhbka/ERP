# Report — Fix lỗi `workPresenceStatusIn should not exist`

## Yêu cầu thay đổi

Sửa lỗi validation khi lọc cột **Làm việc** trên trang Quản lý hồ sơ: `property workPresenceStatusIn should not exist`.

## Việc đã thực hiện

1. **`apps/api/src/employees/dto/employee.dto.ts`**
   - Thêm `@IsArray()` cho `statusIn`, `employmentStatusIn`, `workPresenceStatusIn` (sau `@Transform`) để validation array enum đúng chuẩn.

2. **Rebuild API**
   - Chạy `pnpm --filter @erp/api build` — xác nhận `workPresenceStatusIn` đã có trong `apps/api/dist/employees/dto/employee.dto.js`.
   - Nguyên nhân gốc: `dev:up` chạy `node dist/main.js` nhưng `dist` cũ (trước khi thêm filter enum), nên `ValidationPipe` với `forbidNonWhitelisted: true` từ chối query param mới.

3. **`scripts/dev-start.sh`**
   - Tự build API khi `dist` thiếu **hoặc** bất kỳ file `src/**/*.ts` nào mới hơn `dist/main.js`, tránh lặp lại lỗi sau khi sửa DTO mà quên build.

4. **Restart dev servers** (`pnpm dev:down` → `pnpm dev:up`).

## Kết quả thực hiện

- **Thành công**: API đã được rebuild và restart với DTO mới; lọc `workPresenceStatusIn` sẽ được chấp nhận thay vì báo `should not exist`.
- **Lưu ý**: Sau mọi thay đổi DTO/controller API, cần build lại (hoặc `pnpm dev:up` — script giờ tự build khi source mới hơn dist).
