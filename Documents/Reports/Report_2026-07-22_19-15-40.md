# Report — 2026-07-22 19:15:40

## Yêu cầu thay đổi

Ghi nhớ và áp dụng vĩnh viễn trên toàn repo ERP:

1. Mọi thay đổi code phải tạo `Report_YYYY-MM-DD_HH-MM-SS.md` trong `Documents/Reports`, mô tả yêu cầu / việc đã làm / kết quả; cập nhật chỉ mục trong `Documents/Reports/Report.md`.
2. Mọi yêu cầu cần sửa code tuân thủ quy trình 8 bước (xác nhận → phương án → chờ chọn → đánh giá rủi ro + bước → chờ duyệt → thực hiện → báo cáo tóm tắt → ghi report).
3. Hỏi đáp đơn giản được trả lời ngay.

Người dùng chọn và xác nhận thực hiện **phương án C**: Cursor rule `alwaysApply` + tạo sẵn chỉ mục `Report.md`.

## Việc đã thực hiện

1. Tạo thư mục `.cursor/rules/` (nếu chưa có).
2. Tạo rule luôn áp dụng:
   - `.cursor/rules/project-change-workflow.mdc`
   - `alwaysApply: true`
   - Nội dung: quy trình 8 bước + bắt buộc ghi report + ngoại lệ hỏi đáp đơn giản.
3. Tạo chỉ mục report:
   - `Documents/Reports/Report.md`
4. Tạo file report này:
   - `Documents/Reports/Report_2026-07-22_19-15-40.md`
5. Cập nhật dòng chỉ mục trong `Report.md` trỏ tới file này.

## Kết quả thực hiện

- **Thành công**: Luật quy trình và report đã được gắn vào repo dưới dạng Cursor rule always-apply; thư mục `Documents/Reports` đã có chỉ mục sẵn sàng cho các thay đổi tiếp theo.
- **Không đụng** runtime app (API/Web/Worker/DB).
- **Follow-up**: Các session Agent sau sẽ tự nhận rule này; khi có yêu cầu sửa code phải tuân thủ quy trình và ghi report.
