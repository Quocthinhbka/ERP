# Report — Đồng bộ form hồ sơ từ server (tab Hợp đồng / Công ty chủ quản)

- Thời điểm: 2026-07-23 11:28:13 (local)

## Yêu cầu thay đổi

Khi chỉnh sửa hồ sơ (tab **Thông tin hợp đồng**), giá trị các trường phải được cập nhật đầy đủ từ máy chủ lên frontend sau tải/lưu. Kiểm tra cụ thể trường **Công ty chủ quản** trên giao diện chỉnh sửa.

## Nguyên nhân phát hiện

1. `EmployeeFormPage` dùng `personalForm.setFieldsValue({ ...data })` — spread toàn bộ phản hồi API (kèm `familyMembers`, `managingCompany` object, …) vào form, không map có kiểm soát.
2. `EmployeeProfileDetail` thiếu `managingCompanyId` / `managingCompany` trong type → khó đồng bộ prop `currentCompany` cho Select.
3. `ManagingCompanySelect` chỉ merge `currentCompany` một lần; sau **Lưu**, nếu danh sách công ty ACTIVE chưa có tên mới hoặc prop chưa cập nhật kịp, Select có thể trống label dù `managingCompanyId` đã đúng.

## Việc đã thực hiện

| File | Thay đổi |
|------|----------|
| `employee-profile-form.ts` | **Mới** — `profileToFormValues()` map tường minh mọi trường scalar + `customValues` + chuẩn hoá ngày |
| `employee-types.ts` | Thêm `managingCompanyId`, `managingCompany` vào `EmployeeProfileDetail` |
| `EmployeeFormPage.tsx` | `applyProfileFromServer()` dùng chung cho tải hồ sơ và sau PATCH; bỏ spread `...data` |
| `DynamicProfileFields.tsx` | `ManagingCompanySelect`: `Form.useWatch('managingCompanyId')`, `useMemo` options, merge công ty từ server + form; `key` remount khi `managingCompany` đổi |

## Kết quả thực hiện

- **Thành công** — sau **Lưu**, form và Select **Công ty chủ quản** nhận đúng `managingCompanyId` + tên từ `managingCompany` trong phản hồi API.
- Không đổi backend; API đã trả đủ quan hệ `managingCompany` qua `getProfile()`.
- Kiểm tra: refresh trang chỉnh sửa hồ sơ → tab Hợp đồng → Công ty chủ quản hiển thị đúng; đổi công ty → Lưu → label cập nhật ngay.
